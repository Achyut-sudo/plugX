DESIGN DOCUMENT 


EXTENSION : .plgx

plugX is a language designed to generate large dynamic documents using dynamically evaluated embedding called 'plug'. It supports modular content accession, rendering using 'contianers'. It also has  a robust error-handling mechanism. With built-in 'standard options' to enhance document content, it simplifies large document creation by utilizing reusable elements with dynamic modification based on requirements.



1. EVALUATION ORDER KEYWORD 
   ------------------------ 

	STAGE 
	-----

		SYNTAX : SOME_OTHER_KEYWORD(s)@STAGE_VALUE or SOME_OTHER_KEYWORD@(STAGE_VALUE)
 
		IT DEFINES THE STAGE OF EXECUTION OR REFERNCES OF OTHER KEYWORDS 
		
		VALID VALUES :
			
			PRE : STATES THAT THE EXECUTION  OR REFERENCES IS MADE BEFORE  THE REFERRED KEYWORD IS EVALUATED 
			POST  : STATES THAT THE EXECUTION OR REFERENCES IS MADE AFTER TJE REFERRED KEYWORD IS EVALUATED 




2. CONTAINER ACCESION CONTROL KEYWORDS 
   ----------------------------------


	ORDER_INDEX 
	-----------
		
		SYNATX : SOME_OTHER_KEYWORD@ORDER_INDEX_VALUE or SOME_OTHER_KEYWORD@(ORDER_INDEX_VALUE) 		
		
		IT DEFINES THE INDEX OF THE REFFERED KEYWORD IN  CONTAINER SCOPE OF THE PARENT OF THE REFERRED CONTAINER 	 

		VALID VALUES : INDEX (INTEGER) OF THE INTENDED CONTIANER TAG , INDEX IN ITS PARENT SCOPE 

		
3. ACCESION KEYWORDS 
   -----------------
	
	1. CONTAINER ACCESION KEYWORDS 
	   ---------------------------

		PARENT
		------

			SYNTAX : PARENT@STAGE_VALUE  or PARENT@(STAGE_VALUE)

			IN NESTED TAG CONTAINER TAG , THIS CONTAINER ACCESSOR KEYWORD  REFERS TO THE PARENT I.E.  CONTAINER TAG THAT WRAPS 
			AROUND THE CURRENT CONTAINER TAG 

		
		SELF 
		----
			SYNTAX : SELF 
		
			REFFERS TO THE SAME CONTAINER TAG, WHICH IS BEING EVALUATED . 

		
		SIBLING  
		-------
			
			SYNTAX : SIBLING@ORDER_INDEX_VALUE@STAGE_VALUE  OR SIBLING@(ORDER_INDEX_VALUE,STAGE_VALUE)
	 
			REFFERS TO THE ORDER_INDEX_VALUEth SIBLING, THE ORDER_INDEX_VALUEth CHILD OF THE PARENT OF THE CURRENT CONTIANER TAG. 

		
		CHILD 
		------
			SYNTAX : CHILD@ORDER_INDEX_VALUE@STAGE_VALUE  OR CHILD@(ORDER_INDEX_VALUE,STAGE_VALUE)

			REFFERS TO THE ORDER_INDEX_VALUEth CHILD CONTAINER TAG OF THE CURRENT CONTAINER TAG.


		GLOBAL
		------
			
			SYNTAX : GLOBAL 
			
			REFFERS TO THE GLOBAL CONTAINER_TAG 

			
		TAG 
		---
			SYNTAX : TAG@TAG_ID_VALUE@STAGE_VALUE OR CHILD@(TAG_ID_VALUE,STAGE_VALUE)
			
			REFFERS TO A CONTAINER TAG DIRECTLY BY THE PROVIDED ATTRIBUTE id  TO THE TAG. 
	 

	2. EXTERNAL ACCESION KEYWORD
	   ------------------------- 

		ENV@USR 
		-------
			SYNTAX : ENV@USR OR ENV@(USR) 
			
			USED FOR ACCESING DATA FROM THE USER ENVIRONMENT VARIABLES 

		ENV@SYS 
		-------
			SYNTAX : ENV@SYS OR ENV@(SYS)
		
			USED FOR ACCESSING DATA FROM THE SYSTEM ENVIRONMENT VARIABLES 

		CONFIG@CONFIG_REF 
		----------------

			SYNTAX : CONFIG@CONFIG_REF 
			
			USED FOR ACCESING  THE CONFIG FILE INCLUDED IN THE DOCUMENT THROUGH 'CONFIG' ATTRIBUTE IN GLOBAL CONTAINER TAG

		IMPORT@IMPORT_ID 
		----------------

			SYNTAX : IMPORT@IMPORT_ID
			
			USED FOR ACCESING OTHER .plgX FILE INCLUDED IN THE DOCUMENT THROUGH 'IMPORT' ATTRIBUTE IN GLOBAL CONTAINER TAG




	3. INTERNAL ACCESSION KEYWORD 
	   --------------------------

		BUILTIN
		------- 
			SYNTAX : BUILTIN 
			
			USED FOR ACCESSION OF BUILTIN ATTRIBUTE AND METHODS, CALLED STANDARD OPTIONS 



4. ACCESSION KEYWORD CHAINING
  ---------------------------
	
	   CONTAINER ACCESSION KEYWORD CHAINING 
	   ------------------------------------

		ALL CONTAINER ACCESSION  KEYWORDS ,EXCEPT GLOBAL KEYWORD CAN BE CHAINED, BUT STAGE EVALUTION ORDER KEYWORD MUST BE DEFINED 
		ONLY
		ONCE AT THE END , WHILE THE CONTAINER ACCESSION CONTROL KEYWORD ORDER_INDEX CAN BE DEFINED AS REQUIRED
		
		FOR EXAMPLE : PARENT.PARENT.SIBLING@3@PRE -> THIRD SIBLING OF THE GRANDPARENT OF  THE CURRENT CONTAINER WITH  EVAULAUTION 
			      STAGE PRE

	   
	   EXTERNAL ACCESION KEYWORD CHAINING
	   ----------------------------------	 

		THE IMPORT EXTENAL ACCESION KEYWORD CAN BE CHAINED WITH CONTAINER  ACCESION KEYWORDS AS DESCRIBED ABOVE . 
 

5.  PLUG
    ---- 

	USAGE : TO INSERT DATA DYNAMICLLAY FROM A FUNCTION CALL OR STATIC VALUE 
	
	SYNTAX  : [ACCESSOR_NAME : ACCESOR_SCOPE : ACESSOR_TYPE : ACCESSOR_PARAMS ]


	ACCESSOR_NAME 
	-------------
		
		NAME OF THE ACCESSOR LIKE NAME OF  A  KEY , NAME OF A METHOD OR  ATTRIBUTE
 
	ACCESSOR_SCOPE 
	--------------


		SCOPE  OF THE ACCESSOR 


		VALID VALUES :
			
		
			+-------------------------------------+-------------------------------------------------------------------------+
 			| VALUE	                              |        DESCRIPTION                                                      |
			+-------------------------------------+-------------------------------------------------------------------------+
                        | BUILTIN                             |  THE PARSER , BUILTIN ATTRIBUTE AND METHODS,AVAILABE AS STANDARD OPTION |   
			| PARENT@STAGE                        |  FLAT/DIRECT TAG ACCESSION , PARENT TAG                                 |
			| SELF                                |  NESTED TAG ACESSION ,  SELF                                            |  
			| SIBLING@ORDER_INDEX@STAGE           |  BESTED TAG ACCESION , SIBLING TAG@DEFAULT = 1                          |
			|	OR                            |                                                                         |
			| SIBLING@(ORDER_INDEX,STAGE)         |                                                                         |
			| CHILD@ORDER_INDEX@STAGE             |  NESTED TAG ACCESION , CHILD TAG OF THE CURREBT TAG                     |
			|	OR                            |                                                                         |
			| CHILD@(ORDER_INDEX,STAGE)           |                                                                         |
                        | GLOBAL                              |  FLAT/DIRECT TAG ACCESION , GLOBAL TAG                                  |
			| TAG@TAG_ID@STAGE                    |  FLAT/DIRECT TAG ACCESION , ANY TAG WITHIN THE DOC WITH ID AS TAG_ID    |
			|	OR                            |                                                                         |
			| TAG@(ORDER_INDEX,STAGE)             |                                                                         |
			| ENV@USR                             |  USER ENVRIONMENT VARIABLES                                             |
			| ENV@SYS                             |  SYSTEM ENVIRONMENT VAORABLES                                           |
			| CONFIG@CONFIG_REF                   |  CONFIG FILE ACCESSION , FOR READING VALUES FROM A LOCAL CONFI FILE     | 
			| IMPORT@IMPORT_REF                   |  IMPORTED  FILE ACCESSION , FOR ACCESSING  ELEMENTS OF THE IMPORT .plgX |
			|                                     |  FILES                                                                  |
			+-------------------------------------+-------------------------------------------------------------------------+



	ACCESSOR_TYPE
	------------- 

		TYPE OF ACCESSION ,  I.E IN TERMS OF REQUIREMENT OF EXECTUION, SINCE THE ACCESSOR CAN ALSO REFER TO AMETHOS TOO, HENCE 
                ACCESSOR_TYPE IS USED TO EXPLICITLY DISTINGUID BETWEEN A METHOD ACCESSOR OR A ATTRIBUTE/KEY ACCESSOR 

		
		VALID VALUES :
			
	
			+---------------------+-------------------------------------------------------------------------------------------+ 
                        | VALUES              | DESCRIPTION                                                                               |
                        +---------------------+-------------------------------------------------------------------------------------------+
                        | COMPUTED            | ACCESSION TYPE , DENOTES THAT THE ACCESSOR IS  METHOD                                     |
			| PROPERTY            | ACCESSION TYPE , DENOTES THAT THE ACCESSOR IS STATIC VALUE , DOES NOT NEED TO BE COMPUTED |
                        +---------------------+-------------------------------------------------------------------------------------------+


	ACCESSOR_PARAMS
	--------------- 
		
		+------------------+-----------------------------------------------------------------------+
                |  VALUE           |   SYNTAX                                                              |
		+------------------+-----------------------------------------------------------------------+		      
                |  MULTI           | (POSITIONAL_VAL1, POSITIONAL_VAL2, KEYWORD1=VAL3,KEYWORD2=VAL4)       | 
		|  SINGLE          | (POSTIONAL_VAL1) OR POSTIONAL_VAL1 OR (KEYWORD1=VAL1) OR KEYWORD1=VAL1|
		+------------------+-----------------------------------------------------------------------+
